# Q1. What are the different types of Exceptions in Java?

## Answer

In Java, exceptions are mainly divided into two categories:

```
Throwable
│
├── Error
│
└── Exception
      │
      ├── Checked Exception
      │
      └── Unchecked Exception (Runtime Exception)
```

---

## 1. Checked Exception

Checked Exceptions are checked at **compile time**.

The compiler forces us to handle them using `try-catch` or `throws`.

### Examples

- IOException
- SQLException
- ClassNotFoundException
- FileNotFoundException

### Example

```java
import java.io.*;

public class Main {

    public static void main(String[] args) {

        try {

            FileReader file =
                    new FileReader("data.txt");

        } catch (IOException e) {

            e.printStackTrace();

        }

    }

}
```

---

## 2. Unchecked Exception (Runtime Exception)

These exceptions occur at runtime.

The compiler does not force us to handle them.

### Examples

- NullPointerException
- ArithmeticException
- ArrayIndexOutOfBoundsException
- NumberFormatException
- IllegalArgumentException

### Example

```java
String name = null;

System.out.println(name.length());
```

Output

```
NullPointerException
```

---

## 3. Error

Errors are serious problems generated by the JVM.

Applications generally should not try to recover from them.

### Examples

- OutOfMemoryError
- StackOverflowError
- VirtualMachineError

---

## Difference

| Checked Exception | Unchecked Exception |
|-------------------|--------------------|
| Checked at compile time | Occurs at runtime |
| Must be handled | Handling is optional |
| Extends Exception | Extends RuntimeException |

---

## Real-world Example

### Checked Exception

Reading a file.

The file may not exist.

---

### Runtime Exception

User enters invalid input.

```
Integer.parseInt("ABC");
```

Throws

```
NumberFormatException
```

---

## Interview Tip

Remember

```
Throwable

↓

Error

↓

Exception

↓

Checked

↓

Unchecked
```

---

## Quick Revision

| Type | Examples |
|------|----------|
| Checked | IOException, SQLException |
| Unchecked | NullPointerException, ArithmeticException |
| Error | OutOfMemoryError, StackOverflowError |

---

## Common Mistakes

❌ RuntimeException is a Checked Exception.

❌ Errors should be handled using try-catch.

# Q2. How do you implement a Custom Exception?

## Answer

A Custom Exception is a user-defined exception.

It is created by extending either:

- Exception (Checked Exception)
- RuntimeException (Unchecked Exception)

---

## Example

```java
class InvalidAgeException extends Exception {

    public InvalidAgeException(String message) {

        super(message);

    }

}
```

---

## Using Custom Exception

```java
public class Main {

    static void validateAge(int age)
            throws InvalidAgeException {

        if (age < 18) {

            throw new InvalidAgeException(
                    "Age must be 18 or above."
            );

        }

        System.out.println("Eligible");

    }

    public static void main(String[] args) {

        try {

            validateAge(15);

        } catch (InvalidAgeException e) {

            System.out.println(e.getMessage());

        }

    }

}
```

---

## Output

```
Age must be 18 or above.
```

---

## Real-world Example

Custom Exceptions are commonly used in Spring Boot applications.

Examples

- UserNotFoundException
- InvalidOTPException
- PaymentFailedException
- OrderNotFoundException
- InsufficientBalanceException

---

## Interview Tip

Whenever business rules need custom validation,

use a Custom Exception.

---

```

↓

Extend Exception

↓

throw

↓

catch
```

## Common Mistakes

❌ Extending Throwable directly.

❌ Forgetting to call `super(message)`.

# Q3. Explain try-catch, throw, throws, and finally.

## Answer

These keywords are used for exception handling in Java.

---

# try

The code that may throw an exception is written inside the `try` block.

Example

```java
try {

    int result = 10 / 0;

}
catch (ArithmeticException e) {

    System.out.println(e.getMessage());

}
```

---

# catch

Used to handle an exception.

Example

```java
catch (Exception e) {

    System.out.println(e.getMessage());

}
```

---

# throw

Used to explicitly throw an exception.

Example

```java
throw new ArithmeticException("Invalid Operation");
```

---

# throws

Used in the method signature to declare that a method may throw an exception.

Example

```java
public void readFile()
        throws IOException {

}
```

---

# finally

The `finally` block always executes,

whether an exception occurs or not.

It is mainly used for resource cleanup.

Example

```java
try {

    System.out.println("Try");

}
catch (Exception e) {

    System.out.println("Catch");

}
finally {

    System.out.println("Finally");

}
```

Output

```
Try

Finally
```

---

## Difference between throw and throws

| throw | throws |
|--------|---------|
| Used to explicitly throw an exception | Declares that a method may throw an exception |
| Used inside method | Used in method signature |
| Throws one exception object | Can declare multiple exceptions |

---

## Example

```java
public void withdraw(double amount)
        throws Exception {

    if (amount <= 0) {

        throw new Exception(
                "Invalid Amount"
        );

    }

}
```

---

## Real-world Example

In Spring Boot,

```java
throw new UserNotFoundException(
        "User not found"
);
```

Global Exception Handler catches it and returns an appropriate HTTP response.

---

## Interview Tip

Remember this:

```
try

↓

Risky Code

catch

↓

Handle Exception

throw

↓

Create Exception

throws

↓

Declare Exception

finally

↓

Cleanup
```

---

## Quick Revision

| Keyword | Purpose |
|----------|----------|
| try | Risky Code |
| catch | Handle Exception |
| throw | Throw Exception |
| throws | Declare Exception |
| finally | Cleanup |

---

## Common Mistakes

❌ `throw` and `throws` are the same.

❌ `finally` executes only when an exception occurs.
