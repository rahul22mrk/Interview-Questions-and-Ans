# Q1. How is rate limiting implemented in your current project?

## Answer

In my project, rate limiting is implemented using **Redis**.

Whenever a user requests an OTP, the application stores a key in Redis with an expiration time (TTL).

If the same user requests another OTP before the TTL expires, the request is rejected.

This prevents users from repeatedly requesting OTPs and protects the application from abuse.

---

## Flow

```
User Requests OTP
        │
        ▼
Check Redis Key
        │
 ┌──────┴──────┐
 │             │
Exists      Doesn't Exist
 │             │
Reject      Generate OTP
 │             │
Return      Store Key in Redis (TTL)
```

---

## Example

```java
String key = "OTP:" + mobileNumber;

if (redisTemplate.hasKey(key)) {
    throw new RuntimeException("Please wait before requesting another OTP.");
}

redisTemplate.opsForValue().set(key, "sent", Duration.ofMinutes(1));
```

---

## Why Redis?

- Extremely fast (In-Memory)
- Supports TTL
- Scalable
- Distributed
- Suitable for Microservices

---

## Real-world Example

OTP Service

```
Rahul requests OTP

↓

Redis stores

OTP_9876543210

↓

Expires after 60 seconds

↓

Another request within 60 sec

↓

Rejected
```

---

## Interview Tip

Mention **Redis + TTL** instead of only saying "we stored a value in Redis."

# Q2. What is a Thread Pool?

## Answer

A Thread Pool is a collection of reusable worker threads.

Instead of creating a new thread for every task, tasks are assigned to existing threads.

After completing a task, the thread returns to the pool and is reused.

---

## Why is it Used?

- Better Performance
- Thread Reuse
- Reduced Thread Creation Cost
- Better Resource Management

---

## Example

```java
ExecutorService executor =
        Executors.newFixedThreadPool(5);

executor.submit(() -> {

    System.out.println(
            Thread.currentThread().getName()
    );

});

executor.shutdown();
```

---

## Real-world Example

Suppose an API receives 10,000 requests.

Creating 10,000 threads is expensive.

Instead,

```
100 Threads

↓

Handle

↓

10,000 Requests

↓

Reuse Threads
```

---

## Interview Tip

Remember

**ExecutorService manages the Thread Pool.**

---

```
↓

Thread Pool

↓

Worker Thread

↓

Execute

↓

Reuse
```

# Q3. How would you handle a failing microservice without breaking the entire application flow?

## Answer

A failing microservice should not stop the entire application.

Common approaches are:

- Retry
- Timeout
- Circuit Breaker
- Fallback

---

## Retry

Retries failed requests automatically.

Useful for temporary failures.

---

## Timeout

Stops waiting after a predefined duration.

Prevents thread blocking.

---

## Circuit Breaker

If a service keeps failing,

the circuit opens,

and requests are not sent until the service recovers.

Popular Library

```
Resilience4j
```

---

## Fallback

Returns a default response if the service is unavailable.

Example

```
Product Service Down

↓

Return

"Product temporarily unavailable."
```

---

## Real-world Flow

```
API

↓

Product Service

↓

Failure

↓

Circuit Breaker

↓

Fallback Response
```

---

## Interview Tip

Mention

Retry + Timeout + Circuit Breaker + Fallback

together.

---

## Quick Revision

```
Retry

↓

Timeout

↓

Circuit Breaker

↓

Fallback
```

# Q4. How would you create 10 threads using ExecutorService and submit tasks?

## Answer

```java
ExecutorService executor =
        Executors.newFixedThreadPool(10);

for (int i = 1; i <= 10; i++) {

    int task = i;

    executor.submit(() -> {

        System.out.println(
                "Task " + task +
                " executed by " +
                Thread.currentThread().getName()
        );

    });

}

executor.shutdown();
```

---

## Output

```
Task 1

↓

pool-1-thread-1

Task 2

↓

pool-1-thread-2

...
```

---

## Interview Tip

Always call

```java
shutdown()
```

after submitting tasks.

---

## Possible Follow-up Questions

- Fixed Thread Pool vs Cached Thread Pool?

# Q5. What are the different ways to create a thread in Java?

## Answer

### 1. Extending Thread Class

```java
class MyThread extends Thread {

    @Override
    public void run() {

        System.out.println("Running");

    }

}

new MyThread().start();
```

---

### 2. Implementing Runnable

```java
class Task implements Runnable {

    @Override
    public void run() {

        System.out.println("Running");

    }

}

new Thread(new Task()).start();
```

---

### 3. Using Lambda

```java
new Thread(() -> {

    System.out.println("Running");

}).start();
```

---

### 4. Using ExecutorService ⭐ (Recommended)

```java
ExecutorService executor =
Executors.newFixedThreadPool(5);

executor.submit(() -> {

    System.out.println("Running");

});

executor.shutdown();
```

---

## Interview Tip

Modern Java applications mostly use

**ExecutorService**

instead of manually creating threads.
