# Q1. Have you used Redis? Where and why did you use it?

## Answer

Yes.

In my project, I used Redis for **OTP Rate Limiting**.

Whenever a user requests an OTP, a key is stored in Redis with a TTL (Time To Live). If the user requests another OTP before the TTL expires, the request is rejected.

This prevents OTP abuse and reduces unnecessary server load.

---

## Project Flow

```
User Requests OTP

        │
        ▼

Spring Boot API

        │
        ▼

Check Redis

        │
 ┌──────┴────────┐
 │               │
Key Exists    Key Not Found
 │               │
Reject        Generate OTP
 │               │
Return        Save Key in Redis
                    │
                    ▼
              TTL = 60 Seconds
```

---

## Why Redis?

- Extremely Fast (In-Memory)
- Supports TTL
- Supports Distributed Applications
- Lightweight
- Easy Integration with Spring Boot

---

## Example

```java
String key = "OTP:" + mobile;

if(redisTemplate.hasKey(key)){
    throw new RuntimeException(
        "Please wait before requesting another OTP."
    );
}

redisTemplate.opsForValue()
             .set(key, "SENT", Duration.ofMinutes(1));
```

# Q2. Why did you use Redis instead of MySQL for OTP Rate Limiting?

## Answer

Redis is much faster than MySQL because it stores data in memory.

For OTP rate limiting, we need:

- Very fast read/write
- Automatic expiry
- High throughput

Redis provides all of these.

MySQL stores data on disk, making it slower for this use case.

---

## Comparison

| Redis | MySQL |
|--------|-------|
| In-Memory | Disk Based |
| Very Fast | Comparatively Slower |
| Built-in TTL | No Built-in TTL |
| Best for Cache | Best for Permanent Data |
| Millions of Operations/sec | Lower Throughput |

---

## Example

OTP expires automatically.

```java
redisTemplate.opsForValue()
             .set(
                 key,
                 value,
                 Duration.ofMinutes(1)
             );
```

No cleanup job is required.

# Q3. What is TTL (Time To Live)?

## Answer

TTL (Time To Live) specifies how long a key should remain in Redis.

After the specified time expires, Redis automatically deletes the key.

---

## Example

```java
redisTemplate.opsForValue()
             .set(
                "OTP:9876543210",
                "123456",
                Duration.ofMinutes(1)
             );
```

After 1 minute,

the key is automatically removed.

---

## Benefits

- No Manual Cleanup
- Automatic Expiration
- Memory Optimization

---

## Real-world Example

↓

# Q5. What data structures does Redis support?

## Answer

Redis is a **Data Structure Store**.

It supports multiple data structures.

| Data Structure | Use Case |
|---------------|----------|
| String | OTP, Tokens |
| Hash | User Profile |
| List | Queues |
| Set | Unique Values |
| Sorted Set | Leaderboards |
| Bitmap | Analytics |
| HyperLogLog | Approximate Counting |
| Stream | Event Streaming |

---

## Example

Store OTP

```java
redisTemplate.opsForValue()
             .set("OTP", "123456");
```

Store User

```java
redisTemplate.opsForHash()
             .put("USER:1", "name", "Rahul");
```
