# Q7. How do you start a thread and get the current thread's name?

## Answer

A thread is started using the `start()` method.

The `start()` method internally creates a new thread and invokes the `run()` method.

To get the current thread name, use:

```java
Thread.currentThread().getName();
```

---

## Example

```java
class MyThread extends Thread {

    @Override
    public void run() {

        System.out.println(
                Thread.currentThread().getName()
        );

    }

}

public class Main {

    public static void main(String[] args) {

        MyThread thread = new MyThread();

        thread.start();

    }

}
```

---

## Output

```
Thread-0
```

---

## Main Thread Example

```java
System.out.println(
        Thread.currentThread().getName()
);
```

Output

```
main
```

---

## Interview Tip

Never call

```java
thread.run();
```

directly.

Always call

```java
thread.start();
```

Because `run()` executes in the current thread, while `start()` creates a new thread.

---

## Quick Revision

```text
start()

↓

New Thread

↓

run()
```

---

## Common Mistakes

❌ Calling `run()` directly.

❌ Assuming `start()` executes the code immediately.

# Q8. Explain the Thread Lifecycle.

## Answer

A Java thread goes through different states during its lifecycle.

```
NEW
 ↓
RUNNABLE
 ↓
RUNNING
 ↓
WAITING / BLOCKED / TIMED_WAITING
 ↓
TERMINATED
```

---

## 1. NEW

Thread object is created but `start()` has not been called.

```java
Thread t = new Thread();
```

---

## 2. RUNNABLE

After calling

```java
t.start();
```

the thread enters the Runnable state.

It is waiting for CPU scheduling.

---

## 3. RUNNING

CPU assigns execution to the thread.

The `run()` method executes.

---

## 4. WAITING

The thread waits indefinitely.

Example

```java
join()
```

or

```java
wait()
```

---

## 5. TIMED_WAITING

The thread waits for a specific duration.

Example

```java
Thread.sleep(5000);
```

---

## 6. BLOCKED

Waiting to acquire a monitor lock.

Occurs when another thread already owns the synchronized block.

---

## 7. TERMINATED

The thread finishes execution.

---

## Diagram

```
NEW

↓

RUNNABLE

↓

RUNNING

↓

WAITING / BLOCKED / TIMED_WAITING

↓

RUNNING

↓

TERMINATED
```

---

## Interview Tip

Many candidates forget **BLOCKED** and **TIMED_WAITING**.

Mention all states.

# Q9. If there are 10 tasks but only 2–3 threads available, how are tasks scheduled internally?

## Answer

When using a Thread Pool, tasks are **not assigned a new thread every time**.

Instead,

- Tasks are placed in a queue.
- Available worker threads pick tasks one by one.
- After completing a task, the thread returns to the pool and processes the next task.

# Q10. What problems can occur if concurrency isn't handled properly?

## Answer

Improper concurrency can lead to multiple issues in an application.

---

## 1. Race Condition

Two threads update the same data simultaneously.

Example

```
Balance = 1000

Thread A Withdraws 500

Thread B Withdraws 700

Final Balance becomes incorrect.
```

---

## 2. Deadlock

Two threads wait for each other's lock forever.

Example

```
Thread A

↓

Lock 1

↓

Waiting for Lock 2
```

```
Thread B

↓

Lock 2

↓

Waiting for Lock 1
```

Both threads wait forever.

---

## 3. Starvation

One thread never gets CPU time because higher-priority threads keep executing.

---

## 4. Livelock

Threads keep responding to each other but never complete the task.

---

## 5. Data Inconsistency

Shared data becomes incorrect because multiple threads update it simultaneously.

---

## 6. Performance Issues

Creating too many threads increases

- CPU usage
- Memory usage
- Context switching

---

## How to Prevent These Problems?

- synchronized
- Lock API
- Atomic Classes
- Concurrent Collections
- ExecutorService
- Thread Pool
- ConcurrentHashMap

---

## Real-world Example

In an online banking application,

two users withdrawing money from the same account simultaneously can result in an incorrect balance if synchronization is not used.
